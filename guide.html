publishing guide

html

-html validator 유효성 검사를 문제없이 통과해야 한다.

-웹 브라우저가 반응형으로 동작할 수 있도록 viewport를 사용한다.

-한 문서에서 동일한 id를 사용하지 않는다.

-레이아웃 영역 구분 시 주석을 작성한다.

-주석표기는 해당 내용의 윗줄에 작성한다.

-모든 요소와 속성은 소문자를 사용한다.

-attribute 값은 큰따옴표(“”)로 묶는다.

-코드의 가독성을 위해 들여쓰기를 사용한다.

-body 태그 안의 모든 요소를 감싸는 태그는 클래스명에 wrap을 부여한다.

css

-css validator 유효성 검사를 문제없이 통과해야 한다.

-html에 따른 css는 외부문서로 연결한다.

-normalize.css는 공통으로 연결되어야 한다.

-공통 선택자 ‘*’ 은 사용하지 않는다.

-클래스명은 해당 내용과 관련되어있는 제목으로 작성한다.

-같은 내용을 가지고 있는 클래스일 경우 제목과 함께 숫자를 작성한다.

-클래스, 아이디명은 숫자, 특수문자로 시작하지 않는다.

-css 최적화를 위해 속성의 값을 축약하여 사용할 수 있을 경우 축약형을 작성한다.

script

-스크립트 연결은 head 태그 안에서 이루어져야 한다.

-html에 따른 script 파일은 외부문서로 연결한다.

img

-이미지 파일 형태 권장 : jpg >　png > gif

-기본 포맷은 jpg를 사용한다.

-로고, 아이콘과 같이 투명한 효과가 필요한 경우 png를 사용한다.

-기타 GIF가 필요한 경우 사용 가능하다.

-이미지의 alt 태그는 반드시 작성한다.
